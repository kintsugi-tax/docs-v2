This checklist outlines the core capabilities your integration needs to validate for Kintsugi Partner Certification. Use this as a guide to ensure your integration is ready for validation.

## Core Checklist Items

These items are required for all integrations:

### Transaction Syncing

-  Orders sync successfully to Kintsugi
-  Refunds sync successfully to Kintsugi (credit notes)
-  Transaction syncing preserves data integrity (amounts, item details, customer info)
-  External IDs are unique and consistent
-  Transaction dates are accurate and properly formatted

<Card title="Credit Notes" icon="link" href="/docs/api-guides/handling-refund-transactions">

### Tax Calculations

-  Tax calculations return accurate rates for test addresses
-  Tax calculations work with full address (city + state + zip)
-  Tax calculations work with state + zip (without city)
-  Tax calculations work with zip code only (fallback scenario)
-  Tax calculations handle multiple jurisdictions correctly
-  Tax calculations apply correct rates for different product types

<Card title="Tax Estimation API" icon="link" href="/reference/api/tax-estimation/estimate-tax">
  Learn how to use the tax estimation endpoint
</Card>

### Address Validation (Optional, recommended)

-  Address errors provide specific, actionable error messages
-  Full address validation (city + state + zip) works correctly
-  Fallback to state + zip when city validation fails
-  Fallback to zip code only when full validation fails
-  Address validation errors are user-friendly

<Card title="Address Validation API" icon="link" href="/reference/api/address-validation/search">
  Learn how to validate addresses
</Card>

### Error Handling

-  API error handling manages rate limits gracefully
-  Retry logic handles transient failures
-  Your integration handles Kintsugi API response changes
-  Error messages are logged appropriately for debugging
-  Failed requests don't cause data loss

<Card title="Error Handling Guide" icon="link" href="/docs/error-handling">
  Learn best practices for error handling
</Card>

### Data Integrity

-  Customer records sync correctly with external IDs
-  Product records sync correctly with external IDs
-  Transaction amounts match source system amounts
-  Currency handling is correct
-  Date/time handling preserves timezone information

### Testing

-  You've tested with multiple state/jurisdiction combinations
-  You've tested with different address formats
-  You've tested refund scenarios (full and partial)
-  You've tested error scenarios (invalid addresses, API failures)
-  You've tested with real transaction data from your platform

-  Subscription billing transactions sync correctly

-  Nexus registration management (if applicable to your platform)

### Platform-Specific Capabilities

**For E-Commerce Platforms:**
-  Checkout flow integrates tax calculations
-  Order confirmation syncs transactions
-  Refund workflow creates credit notes

**For Accounting Software:**
-  Invoice creation triggers tax calculations
-  Quote-to-cash cycle works end-to-end

-  Multi-channel order tracking works
-  Inventory-aware tax calculations are correct
-  Stock adjustment tax implications handled

## Preparing for Validation

Before starting the validation process:

1. **Complete the Core Checklist**: Ensure all required items are working
2. **Review Optional Items**: Identify which advanced items apply to your platform
3. **Test Thoroughly**: Use real data from your platform, not just test cases
4. **Document Edge Cases**: Note any platform-specific behaviors or limitations
5. **Prepare Test Scenarios**: Have test cases ready that demonstrate each capability

<Note>
**Don't worry about perfection**: The validation process is collaborative. If we find issues, we'll work with you to resolve them. The goal is to ensure your integration works reliably for your customers.
</Note>
