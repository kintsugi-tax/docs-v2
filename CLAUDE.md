# CLAUDE.md — Kintsugi Docs (docs-v2)

This file provides context for AI assistants (Claude Code, Cursor, etc.) working in this repository.

## Project Overview

This is the Kintsugi documentation site — a [Mintlify](https://mintlify.com)-based docs platform for Kintsugi's automated sales tax API. It contains:

- **Human-authored guides** in `docs/` (~38 MDX files)
- **Auto-generated API reference** in `reference/` (~356 MDX files, regenerated by CI)
- **Interactive API playgrounds** in `recipes/` (6 MDX files)
- **OpenAPI specs** at the repo root (4 JSON files)

## Local Development

```bash
npm i -g mintlify     # Install CLI globally (one-time)
mintlify dev          # Start dev server at http://localhost:3000
mintlify install      # Re-install deps if dev server fails to start
```

The dev server requires `docs.json` to be in the working directory (repo root).

## Content Structure

Navigation is defined entirely in `docs.json` under `navigation.versions[0].tabs`. The site has three tabs:

1. **Documentation** — `docs/` hand-authored guides
   - Getting Started, Guides, API Guides, API Guides for Partners, Partner Certification, Advanced, SDKs, MCP
2. **API Lab** — `recipes/` interactive playgrounds
3. **API Reference** — `reference/api/` auto-generated from `openapi.json`
4. **API Reference - Partners** — `reference/partners/` auto-generated; also drives MCP endpoints

## Writing and Editing Docs

### MDX Page Frontmatter

All guide pages use:
```yaml
---
title: "Page Title"
description: "SEO-friendly description"
---
```

API reference pages use:
```yaml
---
openapi: post /v1/transactions
---
```

Recipe pages add:
```yaml
---
mintlify:
  playground:
    display: interactive
---
```

### Available Mintlify Components

Use these JSX components directly in MDX (no imports needed for built-ins):

| Component | Usage |
|---|---|
| `<Note>` | Key info callouts |
| `<Warning>` | Critical warnings |
| `<Tip>` | Helpful hints |
| `<Check>` | Success confirmations |
| `<Info>` | Informational callouts |
| `<CardGroup cols={2\|3}>` + `<Card>` | Grid of cards with icon/href |
| `<Steps>` + `<Step title="...">` | Numbered procedures |
| `<Tabs>` + `<Tab title="...">` | Tabbed content |
| `<Accordion title="...">` + `<AccordionGroup>` | Collapsible sections |

### Adding a New Guide Page

1. Create `docs/<section>/my-page.mdx` with required frontmatter
2. Add the path to `docs.json` under the appropriate group in the `Documentation` tab
3. Paths in `docs.json` are relative, no `.mdx` extension (e.g., `"docs/guides/my-page"`)

## Automated API Doc Pipeline

The CI workflow (`.github/workflows/update-api-docs.yml`) runs daily and on pushes to scripts:

1. Downloads fresh specs from live APIs:
   - Customer API: `https://api-doc.trykintsugi.com/openapi.json` → `openapi-customer.json`
   - Partners API: `https://api.trykintsugi.com/openapi.json` → `openapi-partners.json`
2. Runs `scripts/create-merged-openapi.py` to produce `openapi-mcp.json` (filtered for MCP)
3. Copies `openapi-customer.json` → `openapi.json`, merging enhanced descriptions from Partners API
4. Runs `@mintlify/scraping` to regenerate `reference/api/` from `openapi.json`
5. Commits changes with `[skip ci]` to avoid loops

**Do not manually edit files in `reference/api/` or `reference/partners/`** — they are auto-generated and will be overwritten by CI.

## MCP Integration

The MCP (Model Context Protocol) server endpoint is at `https://kintsugi.mintlify.app/mcp`.

`openapi-mcp.json` is the filtered spec that powers MCP. It is built from endpoints explicitly listed in the "API Reference - Partners" tab. To add an endpoint to MCP:
1. Ensure the endpoint is in `reference/partners/` and listed in `docs.json` under the "API Reference - Partners" tab
2. Re-run the CI pipeline (or `scripts/create-merged-openapi.py` locally)
